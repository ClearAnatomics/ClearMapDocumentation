widgets#

Custom widgets for the ClearMap GUI.

This module provides reusable Qt widgets and compound controls used throughout the ClearMap graphical interface, including:

  • OrthoViewer — orthogonal slice viewer for 3-D images.

  • ProgressWatcher — progress monitoring with log-file polling.

  • TwoListSelection — dual-list item picker (available ↔ selected).

  • CheckableListWidget — list with per-item checkboxes.

  • DataFrameWidget — simple table display for pandas DataFrames.

  • WizardWidget — base class for multi-step dialogs built from .ui files.

  • PatternDialog — file-pattern wizard for tiled acquisitions.

  • SamplePickerDialog — experiment-folder picker with group support.

  • LandmarksSelectorDialog — 3-D landmark selection for registration.

  • StructurePickerWidget / StructureSelector — atlas region tree browser.

  • PerfMonitor — CPU / RAM / GPU usage bars.

  • ExtendableTabWidget — tab widget with an optional (+) button.

  • FileDropListWidget — list that accepts file drag-and-drop.

  • LandmarksWeightsPanel — slider panel for landmark weight tuning.

  • GraphFilterList — AND/OR filter chain for graph properties.

  • ComparisonsWidgetAdapter — group-comparison checkbox builder.

  • GroupsWidgetAdapter — toolbox-based sample-group editor.

  • NProcessesWidget — label + spin box for n_processes.

  • BlockProcessingWidget — grouped controls for block-processing parameters.

  • ClickableFrame — QFrame that emits a clicked signal.

class LandmarksWeightsPanel(parent: QWidget = None, value_to_model=None, model_to_value=None)[source]#

Bases: QFrame

Compact panel that renders one row per landmark-params file.

Each row contains a label, a 0–100 slider, and a value display.

Public API:

  • set_items() — rebuild rows from names and optional initial weights.

  • get_weights() — return current slider values (0–100) as integers.

  • set_weights() — set slider values without rebuilding rows.

  • valueChangedConnect() — Qt-like hook for external binding.

Optional transforms can be supplied to map between slider value (0–100) and model value (float). Defaults are identity; keep scaling (e.g. exp) in the controller.
